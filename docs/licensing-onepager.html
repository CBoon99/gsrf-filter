Overview

Zero Overshoot is available under three licensing models, each structured for different stages of evaluation and deployment. All production use requires a commercial license.

Evaluation

Purpose: For teams evaluating Zero Overshoot for potential production deployment.

Includes:

  • Access to technical documentation and papers
  • Demo use (no rights granted)
  • Initial consultation on system suitability
  • Written assessment of whether Zero Overshoot would provide meaningful safety criteria for your system

Limitations:

  • No production deployment rights
  • No redistribution or resale
  • Evaluation use only

Production

Purpose: For teams deploying Zero Overshoot in a production system.

Includes:

  • Production deployment rights for a single system or defined scope
  • Integration support and guidance
  • Reference implementation assistance
  • Parameter selection and constraint configuration support
  • Validation and testing methodology guidance

Requirements:

  • System-specific integration work
  • Validation of integrated system
  • Commercial license agreement

Enterprise

Purpose: For organisations embedding Zero Overshoot in a product or platform, or deploying across multiple systems.

Includes:

  • Production deployment rights for defined scope (multiple systems, product embedding, platform integration)
  • Priority integration support
  • Custom integration guidance
  • Ongoing support and updates
  • Resale/redistribution rights (subject to agreement terms)

Requirements:

  • Enterprise license agreement
  • Defined scope and use cases
  • Commercial terms negotiation

What Is NOT Included

  • Open-source rights or public distribution
  • Unlimited redistribution without agreement
  • Guaranteed system performance or uptime
  • Certification or regulatory approval (you are responsible for your system's certification)
  • Generic drop-in library (integration requires system-specific work)

Evaluation Flow

  1. Initial contact: Email info@boonmind.io with your system overview and evaluation needs.
  2. System assessment: I'll review your system and provide an assessment of whether Zero Overshoot would provide meaningful safety criteria.
  3. Documentation: You'll receive relevant technical documentation and integration guidance.
  4. Integration planning: If proceeding, we'll plan integration approach, bound configuration, and validation methodology.
  5. License agreement: For production deployment, we'll execute a commercial license agreement defining scope, rights, and obligations.
  6. Integration support: I'll provide integration guidance, reference implementation support, and validation assistance.

Assumptions & Envelope

Zero Overshoot is designed to enforce bounded behaviour within a defined envelope under stated assumptions. When correctly configured, it can provide deterministic boundedness criteria relative to declared system constraints and update limits.
