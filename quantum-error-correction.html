Why this matters

BP is quantum computing's only viable path to real-time error correction. It's also broken.

Why BP is unavoidable

Minimum-weight perfect matching (MWPM) — the current standard — has superlinear computational cost. At distance 15, 20, 30 it becomes too slow for real-time hardware. Every serious quantum hardware team (IBM, Google, IonQ) is moving to belief propagation because it's linear time and massively parallelisable. There is no other viable decoder for production quantum computing.

Why BP fails on surface codes

Surface code Tanner graphs contain length-4 and length-6 cycles. When BP iterates on cyclic graphs, messages reinforce incorrect beliefs through those cycles — the decoder oscillates between wrong answers without converging. This isn't a tuning problem. It's structural. Every team working on BP decoders has hit this wall.

What everyone else does

Current approaches patch BP after it fails: ordered statistics post-processing (OSD) runs a heavy post-processing step. Neural networks retrain per noise model. Modified schedules change message order. All of these modify the decoder itself — meaning every change requires revalidation and retraining.

What GSRF does instead

GSRF shapes the initial log-likelihood ratios before BP starts iterating. The decoder runs completely unmodified on the shaped inputs. No retraining. No revalidation when the noise model changes. No modification to the decoder. The same parameter set works across different code distances and noise levels without retuning.

The key architectural difference: Every existing improvement to BP modifies the decoder itself. GSRF is the only approach that improves convergence from the outside — as a separate preprocessing layer that the decoder never needs to know about. This is why the same GSRF parameters that work on surface codes also work on industrial actuators and distributed validation networks. The mathematics is universal.

The mechanism

Why GSRF fixes BP convergence

BP fails on surface codes because initial log-likelihood ratios (LLRs) — the soft information about which qubits are likely to have errors — enter the decoder in a flat, uninformative state. When the graph has cycles, BP amplifies whatever structure it finds in that initial state. If the initial state is flat, BP amplifies noise and gets stuck.

GSRF reshapes the reliability landscape of those initial LLRs before the decoder sees them. It applies a gradient-stabilised transformation that: (a) pulls each LLR toward a calibrated equilibrium based on the median reliability across all edges, and (b) applies a bounded memory term that prevents any single LLR from dominating. The result is a shaped initial state where BP has a more useful signal to work with — so it converges faster and to better solutions.

Critically, this happens once before iteration begins. The BP decoder runs completely unchanged on the shaped LLRs. No messages are modified. No schedule is altered. From BP's perspective, it just received better initial information.

The ensemble mode goes further: run GSRF preprocessing N times with small controlled perturbations, run BP N times in parallel, pick the best output by syndrome consistency. At N=5, ensemble GSRF achieves additional improvement beyond single-pass without increasing latency — the N runs are parallel.

Implications for the field

What this means for quantum computing progress

Every quantum computing milestone from here requires better error correction at larger code distances. Google's Willow chip targets d=7. IBM's roadmap goes to d=15 and beyond. At those distances, MWPM is too slow and unassisted BP success rates are in the 30–50% range. Something has to fill that gap.

Current approaches all modify the decoder — which means every hardware generation requires re-engineering and revalidation of the decoder itself. GSRF is external, which means it adapts to new hardware, new noise models, and new code distances without decoder changes.

We estimate GSRF-class preprocessing could accelerate the timeline to fault-tolerant quantum computing by 2–4 years by resolving the BP convergence bottleneck without the engineering overhead of decoder modification. This is not a claim about quantum hardware readiness — it is a claim about removing a known software bottleneck that currently blocks progress.
